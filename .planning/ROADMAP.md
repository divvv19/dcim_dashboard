# Project Roadmap

**Goal:** Upgrade DCIM Dashboard features to provide advanced asset management, environmental tracking, power chain analysis, and robust security integrations.

## Phase 1: Real-time Backend Integration & Core Environmental Monitoring
- **Goal:** Connect the mock backend to real hardware protocols and set up historical databases.
- **Requirements:** REQ-1.1, REQ-1.2, REQ-1.3, REQ-1.4
- **Verification:** 
  - [ ] Backend fetches external protocols successfully.
  - [ ] Database accepts persistent historical inputs.
  - [ ] Alerts are effectively broadcast to the user frontend.
- **Status:** Not Started

## Phase 2: Power Chain & Efficiency Management
- **Goal:** Analyze tracked data into meaningful efficiency metrics including PUE and Outlet level insights.
- **Requirements:** REQ-2.1, REQ-2.2, REQ-2.3
- **Verification:** 
  - [ ] Valid PUE calculation appears on the dashboard.
  - [ ] Historical Power graphs correspond to database points.
- **Status:** Not Started

## Phase 3: Comprehensive Asset & Capacity Management
- **Goal:** Upgrade "Rack Designer" with IT asset databases and detailed spatial/wiring maps.
- **Requirements:** REQ-3.1, REQ-3.2, REQ-3.3
- **Verification:** 
  - [ ] Assets have proper models/metadata attached.
  - [ ] Rack front and rear elevations accurately reflect IT database.
  - [ ] Port-to-Port tracking flows cleanly without errors.
- **Status:** Not Started

## Phase 4: Advanced Visualization & Analytics
- **Goal:** Render a modern 3D server room with overlapping analytical representations like heatmaps.
- **Requirements:** REQ-4.1, REQ-4.2, REQ-4.3
- **Verification:** 
  - [ ] 3D Floorview correctly renders environment state.
  - [ ] Heatmaps successfully visualize thermal hotspots.
  - [ ] Custom dashboard successfully saves and loads component layout.
- **Status:** Not Started

## Phase 5: Security & Access Control
- **Goal:** Add physical data center security measures alongside software boundary scopes.
- **Requirements:** REQ-5.1, REQ-5.2, REQ-5.3
- **Verification:** 
  - [ ] Door status triggers alert logging correctly.
  - [ ] RTSP streams connect and render cleanly.
  - [ ] Admin routes prevent View-Only execution attempts.
- **Status:** Not Started
