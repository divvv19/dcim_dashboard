# Plan 02-02: Live PUE Calculation

## Objective
Dynamically calculate Facility PUE inside the backend server and render the true effectiveness rating in the `HomeView` metric widget rather than using the `1.30` mock.

## Requirements
- `server.js` calculates IT Power (sum of PDU active wattages) and computes Facility Power using a simulated Cooling Tax.
- PUE Math = Facility Power / IT Power.
- Append PUE inside the main `dashboard:update` socket snapshot (`stateStore.system.pue`).
- Render `pue` dynamically in the `HomeView` gauge inside `DCIM.jsx`.

## Tasks
1. Edit `server.js` interval loop: create variables for IT Power and Facility Power.
2. Edit `server.js` Influx DB injection: attach `.floatField('pue', stateStore.system.pue)` to the telemetry point.
3. Edit `src/hooks/useRealtimeData.js` if necessary to ensure `system` object correctly persists.
4. Edit `src/DCIM.jsx` inside the `HomeView` to map the `system.pue` variable into the Circular Gauge value.

## Verification

- Start the frontend.
- PUE value should fluctuate organically (e.g. 1.25 -> 1.34) instead of remaining completely static at 1.30.
