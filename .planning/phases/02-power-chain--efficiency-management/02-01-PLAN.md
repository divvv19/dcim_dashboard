# Plan 02-01: Granular PDU Telemetry

## Objective
Upgrade the backend to simulate and track 24 discrete outlet circuits for `pdu1` and `pdu2`, and visualize this granular load density inside the `PDUView` frontend components.

## Requirements
- Each PDU in `server.js` state store must be extended with an `outlets: []` array containing 24 distinct load configurations.
- The interval polling hook must randomly increment/decrement the outlet current values slightly.
- The `PDUView` component in `DCIM.jsx` must map these outlets out as a scrolling LED array (e.g. green for normal load, yellow for high load).

## Tasks
1. Edit `server/server.js` to initialize 24 simulated circuits under `stateStore.pdu1.outlets` and `stateStore.pdu2.outlets`.
2. Update the `server.js` interval logic to mutate the mock currents on these circuits.
3. Edit `src/DCIM.jsx` and restructure the `PDUView` container to accommodate an `<div className="overflow-y-auto">` layout rendering the circuits.
