# Phase 1: Real-time Backend Integration & Core Environmental Monitoring
## Plan 1.3: Alerting System

<objective>
Implement backend logic to detect threshold breaches (e.g., high temp, water leak) and push real-time alerts to the frontend with distinct severity levels.
</objective>

<requirements>
- REQ-1.3: Create a backend threshold alerting system triggering live frontend toasts.
</requirements>

<tasks>
  <task id="1">
    <description>Create an `alertManager` function in `server/server.js` that compares current polled data (`envData`, `upsData`, etc.) against defined safe threshold constants (e.g., Temp > 28C, waterLeak === true).</description>
    <validation>The function reliably evaluates true/false boundary conditions.</validation>
  </task>
  <task id="2">
    <description>Add a distinct Socket.IO event emitter in the backend (`io.emit('system_alert', { severity, message })`) that triggers whenever the `alertManager` detects a newly breached threshold.</description>
    <validation>The backend emits an appropriately formatted alert object when a mock condition is forced.</validation>
  </task>
  <task id="3">
    <description>Update `src/DCIM.jsx` to listen for the `system_alert` Socket event and utilize the existing UI Toast notification overlay system to display them, color-coded by severity (e.g., Critical=Red).</description>
    <validation>Frontend automatically shows toasts when the backend emits a system alert.</validation>
  </task>
</tasks>

<context>
- The Alerting system can be run in Wave 2 parallel to the Sensor expansion since they operate on independent functions. 
</context>

<must_haves>
- The backend analyzes polling data automatically, pushing alerts without frontend polling.
- Alerts contain specific error data and distinct severity rankings.
- The frontend renders these alerts natively inside the Toast UI.
</must_haves>
