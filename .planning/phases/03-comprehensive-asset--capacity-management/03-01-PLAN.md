# Plan 03-01: IT Asset Database & Model Library

**Phase:** 03 — Comprehensive Asset & Capacity Management
**Requirement:** REQ-3.1 — Migrate IT asset mapping to a database holding metadata (Model, Serial, IP, Owner)
**Dependencies:** None (foundational for 03-02 and 03-03)

---

## Goal

Replace the hardcoded 5-item `assetLibrary` array with a rich model library and instance-level asset database. The backend stores asset models (with port definitions) and placed asset instances. The frontend fetches and renders a searchable catalog. Placed assets carry full metadata and unique instance IDs.

### Task 2: Backend — Asset Instance Store + REST API

**File:** `server/server.js`

Add an in-memory `assetStore` (array of placed instances) and REST endpoints:

```js
let assetStore = []; // Array of { assetId, modelId, name, serialNumber, assetTag, ipAddress, owner, status, rackSlotU, notes }
let nextAssetId = 1;

// GET /api/models — return modelLibrary
app.get('/api/models', (req, res) => res.json(modelLibrary));

// GET /api/assets — return all placed assets
app.get('/api/assets', (req, res) => res.json(assetStore));

// POST /api/assets — place a new asset instance
app.post('/api/assets', express.json(), (req, res) => {
    const asset = { assetId: nextAssetId++, ...req.body, status: req.body.status || 'Active' };
    assetStore.push(asset);
    io.emit('assets:update', assetStore);
    res.status(201).json(asset);
});

// PUT /api/assets/:id — update asset metadata
app.put('/api/assets/:id', express.json(), (req, res) => { ... });

// DELETE /api/assets/:id — remove asset
app.delete('/api/assets/:id', (req, res) => { ... });
```

Add `express.json()` middleware globally (`app.use(express.json())`).

### Task 6: Update handleDrop to Create Asset Instances

**File:** `src/DCIM.jsx`

When an asset is dropped onto the rack:
1. `POST /api/assets` with `{ modelId, name: model.modelName, rackSlotU: index }`
2. On success, store the returned `assetId` in the `rackItems` slot entry
3. The slot entry gains: `{ ...model, assetId, type: 'anchor'/'filled', anchorId }`

When removed (`handleRemove`):
1. `DELETE /api/assets/:id` 
2. Clear the slot entries
