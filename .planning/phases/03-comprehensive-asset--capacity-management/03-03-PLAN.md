# Plan 03-03: Port-to-Port Connection Tracking & Cable Tracing

**Phase:** 03 — Comprehensive Asset & Capacity Management
**Requirement:** REQ-3.3 — Enable mapping and traversal of network connections and power cable connections between rack assets
**Dependencies:** Plan 03-01 (asset instances with IDs), Plan 03-02 (port rendering with clickable squares)

---

## Goal

Allow users to create port-to-port connections between rack assets, persist them in the backend, and visualize them as both a structured list and SVG paths within the rack elevation. A "Trace" action highlights the connected ports with a glow animation.

---

## Tasks

### Task 1: Backend — Connection Store & REST API

**File:** `server/server.js`

Add an in-memory `connectionStore` and REST endpoints:

```js
let connectionStore = [];
let nextConnId = 1;

// GET /api/connections
app.get('/api/connections', (req, res) => res.json(connectionStore));

// POST /api/connections — create a new connection
app.post('/api/connections', express.json(), (req, res) => {
    const { srcAssetId, srcPortLabel, dstAssetId, dstPortLabel, cableType, color, label } = req.body;
    
    // Validate: check port not already connected
    const existing = connectionStore.find(c =>
        (c.srcAssetId === srcAssetId && c.srcPortLabel === srcPortLabel) ||
        (c.dstAssetId === srcAssetId && c.dstPortLabel === srcPortLabel) ||
        (c.srcAssetId === dstAssetId && c.srcPortLabel === dstPortLabel) ||
        (c.dstAssetId === dstAssetId && c.dstPortLabel === dstPortLabel)
    );
    if (existing) return res.status(409).json({ error: 'Port already connected' });
    
    const conn = { connectionId: nextConnId++, srcAssetId, srcPortLabel, dstAssetId, dstPortLabel, 
                   cableType: cableType || 'copper', color: color || '#06b6d4', label: label || '' };
    connectionStore.push(conn);
    io.emit('connections:update', connectionStore);
    res.status(201).json(conn);
});

// DELETE /api/connections/:id
app.delete('/api/connections/:id', (req, res) => {
    connectionStore = connectionStore.filter(c => c.connectionId !== parseInt(req.params.id));
    io.emit('connections:update', connectionStore);
    res.sendStatus(200);
});
```

### Task 3: Frontend — "Connect Mode" Port Selection

**File:** `src/DCIM.jsx` — `RackDesignerView`

Add a two-click connection workflow:

```js
const [connectMode, setConnectMode] = useState(null);
// connectMode = null | { assetId, portLabel, portType }

const handlePortClick = (assetId, portLabel, portType) => {
    if (!connectMode) {
        // First click: select source port
        setConnectMode({ assetId, portLabel, portType });
        showToast(`Source: ${portLabel} selected. Click destination port.`, 'info');
    } else {
        // Second click: complete connection
        if (connectMode.assetId === assetId && connectMode.portLabel === portLabel) {
            setConnectMode(null); // Cancel if same port
            return;
        }
        // POST connection
        fetch('http://localhost:5000/api/connections', {
            method: 'POST',
            headers: { 'Content-Type': 'application/json' },
            body: JSON.stringify({
                srcAssetId: connectMode.assetId,
                srcPortLabel: connectMode.portLabel,
                dstAssetId: assetId,
                dstPortLabel: portLabel,
                cableType: connectMode.portType === 'PWR' ? 'power' : 'copper'
            })
        })
        .then(r => { if (!r.ok) throw new Error(); return r.json(); })
        .then(() => { showToast('Connection created!', 'success'); setConnectMode(null); })
        .catch(() => showToast('Port already connected.', 'error'));
    }
};
```

Port squares gain an `onClick` handler and visual feedback for connect mode:

```jsx
<div
    key={pi}
    onClick={(e) => { e.stopPropagation(); handlePortClick(item.assetId, port.label, port.type); }}
    className={`w-3 h-3 rounded-sm border cursor-pointer transition-all
        ${isConnected(item.assetId, port.label) ? 'ring-2 ring-cyan-400 ring-offset-1 ring-offset-slate-900' : ''}
        ${connectMode?.assetId === item.assetId && connectMode?.portLabel === port.label ? 'ring-2 ring-yellow-400 animate-pulse' : ''}
        ${portColorMap[port.type]} border-white/20 hover:scale-150 hover:z-20`}
    title={`${port.label} (${port.type})`}
/>
```

### Task 6: Frontend — SVG Cable Paths (Same-Rack)

**File:** `src/DCIM.jsx` — inside the rack Card

Overlay an SVG canvas on the rack column for connections where both assets are in the same rack:

```jsx
<svg className="absolute inset-0 pointer-events-none z-20" style={{ width: '100%', height: '100%' }}>
    {connections.filter(c => /* both assets in this rack */).map(conn => {
        const srcY = getPortY(conn.srcAssetId, conn.srcPortLabel);
        const dstY = getPortY(conn.dstAssetId, conn.dstPortLabel);
        const midX = -30; // curve to the left of the rack
        return (
            <path key={conn.connectionId}
                d={`M 0 ${srcY} C ${midX} ${srcY}, ${midX} ${dstY}, 0 ${dstY}`}
                stroke={conn.color} strokeWidth={2} fill="none" opacity={0.6}
                className={tracedConnection === conn.connectionId ? 'animate-pulse opacity-100' : ''}
            />
        );
    })}
</svg>
```

The `getPortY` helper calculates the Y pixel coordinate from the asset's rack slot position and port index.
