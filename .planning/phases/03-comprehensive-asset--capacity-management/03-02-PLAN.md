# Plan 03-02: Front & Rear Rack Elevation Views

**Phase:** 03 — Comprehensive Asset & Capacity Management
**Requirement:** REQ-3.2 — Introduce front and rear rack elevation views detailing dimension slots and limits
**Dependencies:** Plan 03-01 (model library with port definitions)

---

## Goal

Add a Front/Rear toggle to the rack column so users can view the rack from either face. Each placed asset renders its face-specific ports as tiny interactive squares. Ports are color-coded by type. The rear view mirrors port order horizontally to match physical reality.

---

## Tasks

### Task 1: Add Front/Rear Toggle UI

**File:** `src/DCIM.jsx` — `RackDesignerView`

Add a `rackFace` state (`'front'` | `'rear'`) and toggle buttons above the rack:

```jsx
const [rackFace, setRackFace] = useState('front');

// Inside the rack Card, before the slot column:
<div className="flex justify-center gap-2 mb-3">
    <button onClick={() => setRackFace('front')}
        className={rackFace === 'front' ? 'bg-cyan-600 ...' : 'bg-slate-700 ...'}>
        Front
    </button>
    <button onClick={() => setRackFace('rear')}
        className={rackFace === 'rear' ? 'bg-cyan-600 ...' : 'bg-slate-700 ...'}>
        Rear
    </button>
</div>
```

The rack title updates to reflect the current face: `"Server Rack 01 (42U) — Front"`.

### Task 2: Render Ports on Asset Blocks

**File:** `src/DCIM.jsx` — inside the asset anchor rendering block

When an asset block is rendered in the rack (the `isAnchor` block at line 575), inject a row of port squares below the asset name:

```jsx
{isAnchor && (
    <div className="absolute top-0 left-0 right-0 z-10 ..." style={{ height: `calc(${item.u * 1.75}rem - 2px)` }}>
        {/* Existing: icon + name + power */}
        <item.icon size={16} ... />
        <span className="...">{item.name}</span>
        
        {/* NEW: Port squares row */}
        <div className="flex gap-[2px] ml-auto">
            {(rackFace === 'front' ? item.frontPorts : item.rearPorts)?.map((port, pi) => (
                <div key={pi}
                    className={`w-3 h-3 rounded-sm border border-white/20 ${portColorMap[port.type]}`}
                    title={`${port.label} (${port.type})`}
                />
            ))}
        </div>
    </div>
)}
```

### Task 3: Define Port Color Map

**File:** `src/DCIM.jsx` — constant near top of file

```js
const portColorMap = {
    ETH:  'bg-cyan-500',
    PWR:  'bg-orange-500',
    FC:   'bg-purple-500',
    SFP:  'bg-green-500',
    MGMT: 'bg-yellow-500',
    USB:  'bg-slate-400',
};
```

### Task 4: Rear View Horizontal Mirror

When `rackFace === 'rear'`, reverse the port array order before rendering:

```js
const ports = rackFace === 'front' 
    ? (item.frontPorts || []) 
    : [...(item.rearPorts || [])].reverse();
```

This ensures the rear view mirrors physical reality (looking at the back of the rack, ports appear reversed).

### Task 5: Port Legend in Rack Metrics Panel

**File:** `src/DCIM.jsx` — inside the "Rack Metrics" Card

Add a port type legend below the existing tips:

<div className="mt-4 space-y-1">

### Task 6: Click-to-Select Asset (Wire Drawer from 03-01)

**File:** `src/DCIM.jsx`

The anchor block gets an `onClick` handler:

```jsx
onClick={() => onSelectAsset(item)}
```

This calls the parent callback which sets `selectedAsset` state, opening the `AssetDrawer` from Plan 03-01. The drawer shows different port lists depending on the currently selected `rackFace`.
